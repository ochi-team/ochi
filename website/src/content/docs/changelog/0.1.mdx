First public milestone focused on end-to-end log ingestion and querying.

### What works

- Loki ingestion protocol over HTTP via `POST /insert/loki/api/v1/push`
- Snappy-compressed request payloads (`Content-Encoding: snappy`)
- Readiness datasource probe at `GET /insert/loki/ready`
- Simple Query API at `POST /query`

### Current limitations

- Only Json Loki ingestion is supported
- Only Snappy compression is supported for ingestion payloads
- Querying uses a very simple JSON language:
  - time range: `start`, `end` (nanoseconds)
  - exact-match tag filters: `tags: [{ key, value }]`
  - exact-match field filters: `fields: [{ key, value }]`

This release is minimal by purpose and validates the core write/read path.
