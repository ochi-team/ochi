<p class="changelog-date">Apr 23, 2026</p>

First public milestone focused on end-to-end log ingestion and querying.

This release is minimal by purpose and validates the core write/read path.

Read [docs](../../reference/0.1/) for more details.

### Ingestion

- Added: Loki ingestion protocol over HTTP via `POST /insert/loki/api/v1/push`, request payload is expected snappy-compressed json (`Content-Encoding: snappy`, `Content-Type: application/json`)

### Query

- Added: Simple Query API at `POST /query`

### UI

- Added: Readiness datasource probe at `GET /insert/loki/ready`, used primarily by Grafana datasource to check if Ochi is ready to accept requests
- Added: Grafana [datasource](https://github.com/ochi-team/ochi-grafana) available, supports Ochi API as a backend
